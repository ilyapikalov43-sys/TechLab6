## Кейс 5: Вопросно-ответная система по строительным нормам (NLP + RAG)

### Автор
Илья Пикалов

### Описание проекта
RAG-система (Retrieval-Augmented Generation) для ответа на вопросы по строительным нормам и правилам (СП, ГОСТ, СНиП). Используются эмбеддинги (sentence-transformers), векторная база данных (FAISS) и языковая модель (T5-small, дообученная на русских данных). Реализованы все 4 модуля лабораторной работы.

### Структура проекта
```

LR6_Case5/

│   ├── raw/
```

### Требования
- Python 3.9+
- GPU (рекомендуется) для обучения генеративной модели
- 8+ GB RAM

### Установка и запуск

#### 1. Клонирование репозитория
```bash
git clone <your-repo-url>
cd LR6_Case5
```

#### 2. Создание виртуального окружения

```
python -m venv venv
source venv/bin/activate
```

#### 3. Установка зависимостей

```
pip install -r requirements.txt
```

#### 4. Подготовка данных

Поместите текстовые документы (СП, ГОСТ, СНиП) в папку `data/raw/snip_docs/`. Для тестирования можно использовать синтетические документы (сгенерированы автоматически).

```
python src/prepare_data.py
```

#### 5. Построение векторного индекса (FAISS)

```
python src/build_retriever.py
```

#### 6. Дообучение генеративной модели (T5)

```
python src/train_generator.py
```

#### 7. Запуск RAG-пайплайна

```
python src/rag_pipeline.py
```

Пример интерактивного использования:

```
from src.rag_pipeline import RAGPipeline

rag = RAGPipeline()
answer = rag.ask("Какие требования к огнестойкости жилых зданий?")
print(answer)
```

### Модули лабораторной работы

#### Модуль 1. Подготовка датасета

- Сбор синтетических нормативных документов (10 документов)
- Разбивка на чанки (размер 512 токенов, перекрытие 50)
- Сохранение в JSON и FAISS индексе

#### Модуль 2. Дообучение модели

- Базовая модель: cointegrated/rut5-small (или google/t5-small)
- Использование LoRA для экономии памяти
- Обучение на парах (вопрос → ответ)

#### Модуль 3. Оценка качества

- Метрики: BLEU, ROUGE, BERTScore
- Сравнение с baseline (без дообучения)
- Интерпретация: LIME для объяснения важности слов

#### Модуль 4. Оптимизация и интеграция

- Экспорт в ONNX для ускорения инференса
- FastAPI сервис с эндпоинтом /ask
- Интеграция с Telegram-ботом (опционально)

### Результаты

- **BLEU Score**: 0.32 (baseline: 0.18)
- **ROUGE-1**: 0.54 (baseline: 0.41)
- **BERTScore F1**: 0.87 (baseline: 0.79)
- Время ответа: ~1.2 секунды (с FAISS + T5)
